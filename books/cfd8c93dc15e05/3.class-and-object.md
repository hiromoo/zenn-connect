## クラスとオブジェクトの違い

前章では、クラスはオブジェクトを作るための設計図だと説明しました。
この章では、クラスとオブジェクトの違いをもう少し詳しく見ていきます。

Java では、クラスを定義し、そのクラスをもとにオブジェクトを作ります。

```java
class Book {
    String title;
    int price;

    void printInfo() {
        System.out.println(title + ": " + price + "円");
    }
}
```

この `Book` はクラスです。
まだ実際の本を表すデータは入っていません。
`Book` という種類のオブジェクトが、`title` と `price` を持ち、`printInfo` という処理を持つことを定義しているだけです。

実際に使うには、`new` を使ってオブジェクトを作ります。

```java
Book book = new Book();
book.title = "Java 入門";
book.price = 2500;

book.printInfo(); // Java 入門: 2500円
```

この `book` がオブジェクトです。
`Book` クラスから作られた実体なので、`Book` のインスタンスとも呼ばれます。

## `new` はオブジェクトを作る

Java でオブジェクトを作るときは、基本的に `new` を使います。

```java
ArrayList<String> names = new ArrayList<>();
```

このコードでは、右側の `new ArrayList<>()` が新しい `ArrayList` オブジェクトを作っています。
左側の `names` は、そのオブジェクトを使うための変数です。

同じクラスから複数のオブジェクトを作ることもできます。

```java
ArrayList<String> names = new ArrayList<>();
ArrayList<String> cities = new ArrayList<>();

names.add("Alice");
cities.add("Tokyo");

System.out.println(names);  // [Alice]
System.out.println(cities); // [Tokyo]
```

`names` と `cities` は、どちらも `ArrayList` オブジェクトですが、別々のオブジェクトです。
そのため、一方に要素を追加しても、もう一方には影響しません。

## フィールドはオブジェクトの状態を表す

クラスの中に定義された変数をフィールドと呼びます。
フィールドは、オブジェクトの状態を表すために使われます。

```java
class User {
    String name;
    int age;
}
```

この `User` クラスでは、`name` と `age` がフィールドです。
`User` オブジェクトごとに、それぞれ別の値を持つことができます。

```java
User user1 = new User();
user1.name = "Alice";
user1.age = 20;

User user2 = new User();
user2.name = "Bob";
user2.age = 25;

System.out.println(user1.name); // Alice
System.out.println(user2.name); // Bob
```

`user1` と `user2` は同じ `User` クラスから作られていますが、フィールドの値は別々です。

## メソッドはオブジェクトの振る舞いを表す

クラスの中に定義された処理をメソッドと呼びます。
メソッドは、オブジェクトの振る舞いを表します。

```java
class User {
    String name;
    int age;

    void introduce() {
        System.out.println("私は" + name + "です。");
    }
}
```

`introduce` メソッドは、`User` オブジェクトが自己紹介する振る舞いを表しています。

```java
User user = new User();
user.name = "Alice";
user.age = 20;

user.introduce(); // 私はAliceです。
```

API リファレンスでメソッドを見るときも、同じように考えます。
`ArrayList` の `add` メソッドは、`ArrayList` オブジェクトに要素を追加する振る舞いです。
`size` メソッドは、現在の要素数を返す振る舞いです。

## オーバーロード

Java では、同じ名前のメソッドを複数定義できます。
ただし、引数の数や型が違っている必要があります。
この仕組みをオーバーロードと呼びます。

```java
class MessagePrinter {
    void print(String message) {
        System.out.println(message);
    }

    void print(String message, int count) {
        for (int i = 0; i < count; i++) {
            System.out.println(message);
        }
    }
}
```

この `MessagePrinter` クラスには、`print` という名前のメソッドが2つあります。
1つ目は `String` を1つ受け取り、2つ目は `String` と `int` を受け取ります。

```java
MessagePrinter printer = new MessagePrinter();

printer.print("Hello");
printer.print("Hello", 3);
```

呼び出すときに渡した引数によって、どちらの `print` メソッドが使われるかが決まります。

`ArrayList` にも、オーバーロードされたメソッドがあります。
たとえば、`add` メソッドは次の2つの形で使えます。

```java
ArrayList<String> names = new ArrayList<>();

names.add("Alice");     // 末尾に追加
names.add(0, "Bob");    // 指定した位置に追加
```

API リファレンスで同じ名前のメソッドが複数出てきたら、引数の数や型の違いに注目しましょう。

## パッケージはクラスを整理する単位

Java では、関連するクラスやインターフェースをパッケージという単位で整理します。
たとえば、`ArrayList` は `java.util` パッケージに含まれています。

API リファレンスでは、クラス名の近くにパッケージ名が表示されます。
`java.util.ArrayList` のように、パッケージ名とクラス名をつなげた名前を完全修飾名と呼びます。

コードで `ArrayList` を使うときは、通常 `import` 文を書きます。

```java
import java.util.ArrayList;

ArrayList<String> names = new ArrayList<>();
```

`import` を書くことで、`java.util.ArrayList` と毎回書かずに、`ArrayList` という短い名前で使えます。

パッケージは、クラス名の衝突を避けたり、関連するクラスを見つけやすくしたりするために使われます。
また、次に説明するアクセス修飾子の「同じパッケージ」という範囲にも関係します。

## アクセス修飾子は見える範囲を決める

Java では、クラス、フィールド、メソッド、コンストラクタにアクセス修飾子を付けることがあります。
アクセス修飾子は、その要素をどこから使えるかを決めるものです。

代表的なアクセス修飾子は次の通りです。

| 修飾子 | 使える範囲 |
| --- | --- |
| `public` | どこからでも使える |
| `private` | 同じクラスの中からだけ使える |
| `protected` | 同じパッケージ、またはサブクラスから使える |
| 何も付けない | 同じパッケージの中から使える |

たとえば、次のクラスでは `name` フィールドを `private` にし、外部からは `getName` メソッドを通して値を取得できるようにしています。

```java
class User {
    private String name;

    User(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

`private` なフィールドは、クラスの外から直接触れません。

```java
User user = new User("Alice");

System.out.println(user.getName()); // Alice
// System.out.println(user.name);   // コンパイルエラー
```

アクセス修飾子を使うことで、外部に公開したいものと、クラスの内部だけで使いたいものを分けられます。
これはカプセル化とも深く関係しています。

API リファレンスに載っているメソッドやコンストラクタは、多くの場合 `public` です。
つまり、そのクラスを使う人に向けて公開されている操作です。
一方、クラスの内部で使われる `private` な実装の詳細は、通常 API リファレンスでは利用者向けの操作としては扱われません。

## `static` はクラスに属することを表す

Java のフィールドやメソッドには、`static` が付くことがあります。
`static` が付いたフィールドやメソッドは、個々のオブジェクトではなく、クラスそのものに属します。

たとえば、`Math.max` メソッドは `static` メソッドです。
`Math` オブジェクトを `new` で作らなくても、クラス名から直接呼び出せます。

```java
int larger = Math.max(10, 20);
System.out.println(larger); // 20
```

一方、`ArrayList` の `add` や `size` は、特定の `ArrayList` オブジェクトに対して呼び出すメソッドです。

```java
ArrayList<String> names = new ArrayList<>();
names.add("Alice");

System.out.println(names.size()); // 1
```

`static` メソッドは、特定のオブジェクトの状態に依存しない処理によく使われます。
API リファレンスで `static` と書かれていたら、「オブジェクトを作って呼ぶのではなく、クラス名から呼ぶもの」と考えると読みやすくなります。

## `static` メソッドから通常のメソッドは直接呼べない

`static` を学ぶときにつまずきやすいのが、`static` メソッドの中から通常のメソッドを直接呼び出そうとしてエラーになるケースです。

```java
class User {
    String name;

    void introduce() {
        System.out.println("私は" + name + "です。");
    }

    static void printSample() {
        introduce(); // コンパイルエラー
    }
}
```

`introduce` は通常のメソッドです。
通常のメソッドは、特定のオブジェクトに対して呼び出されます。
この例では、`introduce` メソッドの中で `name` フィールドを使っていますが、`name` は `User` オブジェクトごとに異なる値です。

一方、`static` メソッドはクラスに属するメソッドです。
`User.printSample()` のように、`User` オブジェクトを作らなくても呼び出せます。
そのため、`static` メソッドの中で `introduce()` とだけ書いても、「どの `User` オブジェクトの `introduce` を呼べばよいのか」が分かりません。

通常のメソッドを呼びたい場合は、先にオブジェクトを作り、そのオブジェクトに対してメソッドを呼び出します。

```java
class User {
    String name;

    User(String name) {
        this.name = name;
    }

    void introduce() {
        System.out.println("私は" + name + "です。");
    }

    static void printSample() {
        User user = new User("Alice");
        user.introduce();
    }
}
```

`main` メソッドも `static` メソッドです。
そのため、`main` の中から自分で作った通常のメソッドを呼び出すときも、基本的にはオブジェクトを作ってから呼び出します。

```java
class Main {
    void hello() {
        System.out.println("Hello");
    }

    public static void main(String[] args) {
        Main main = new Main();
        main.hello();
    }
}
```

`static` メソッドから通常のメソッドを直接呼べない理由は、通常のメソッドが「どのオブジェクトに対する操作なのか」を必要とするためです。

## コンストラクタは初期化を行う

オブジェクトを作るときに呼び出される特別な処理をコンストラクタと呼びます。
コンストラクタは、オブジェクトの初期化に使われます。

```java
class User {
    String name;
    int age;

    User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void introduce() {
        System.out.println("私は" + name + "です。");
    }
}
```

このクラスでは、`User(String name, int age)` がコンストラクタです。
`new User("Alice", 20)` と書くと、このコンストラクタが呼び出されます。

```java
User user = new User("Alice", 20);
user.introduce(); // 私はAliceです。
```

`ArrayList` の API リファレンスにも、コンストラクタの一覧があります。
たとえば、次のようなコンストラクタがあります。

```java
ArrayList<String> list1 = new ArrayList<>();
ArrayList<String> list2 = new ArrayList<>(100);
```

1つ目は、空のリストを作ります。
2つ目は、初期容量を指定してリストを作ります。

API リファレンスでコンストラクタを見ると、「このクラスのオブジェクトをどのように作れるか」が分かります。

## `this` は自分自身のオブジェクトを表す

クラスの中で `this` と書くと、現在のオブジェクト自身を表します。

```java
class User {
    String name;

    User(String name) {
        this.name = name;
    }
}
```

この例では、コンストラクタの引数 `name` とフィールド `name` の名前が同じです。
`this.name` と書くことで、「このオブジェクトのフィールド `name`」を表しています。

`this` は自分でクラスを書くときによく使います。
API リファレンスを読むだけなら頻出ではありませんが、クラスの内部で状態を扱う考え方を理解するうえで役に立ちます。

## API リファレンスで確認するポイント

クラスとオブジェクトの考え方を知ると、API リファレンスでは次の情報が読みやすくなります。

- クラス名: どのような種類のオブジェクトを表すか
- コンストラクタ: オブジェクトをどのように作るか
- メソッド: オブジェクトに対して何ができるか
- パッケージ名: どのまとまりに属するクラスか
- アクセス修飾子: どこから使える API なのか
- `static` メソッド: クラス名から直接呼び出せる操作は何か
- フィールド: クラスが公開している値や定数は何か

`ArrayList` のページを見ると、`ArrayList` というクラスからオブジェクトを作り、そのオブジェクトに対して `add`、`get`、`remove`、`size` などのメソッドを呼び出せることが分かります。

## この章のまとめ

この章では、クラスとオブジェクトについて学びました。

- クラスはオブジェクトの設計図
- オブジェクトはクラスから作られた実体
- フィールドはオブジェクトの状態を表す
- メソッドはオブジェクトの振る舞いを表す
- オーバーロードにより、同じ名前で引数の違うメソッドを複数定義できる
- パッケージは、関連するクラスやインターフェースを整理する単位
- アクセス修飾子は、クラス、フィールド、メソッドなどをどこから使えるかを決める
- `static` が付いたフィールドやメソッドは、オブジェクトではなくクラスに属する
- `static` メソッドから通常のメソッドを呼ぶには、対象となるオブジェクトが必要
- コンストラクタはオブジェクトの初期化を行う

次の章では、Java API リファレンスでよく見る `extends` や `implements` を理解するために、継承とポリモーフィズムを学びます。
