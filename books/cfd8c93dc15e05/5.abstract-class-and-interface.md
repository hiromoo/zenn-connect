## `abstract` と `interface`

Java API リファレンスを読んでいると、`abstract` や `interface` という言葉が出てきます。

`ArrayList` のクラス宣言にも、次のように書かれています。

```java
public class ArrayList<E>
extends AbstractList<E>
implements List<E>, RandomAccess, Cloneable, Serializable
```

ここには、2つの重要な関係が含まれています。

- `ArrayList` は `AbstractList` を継承している
- `ArrayList` は `List` などのインターフェースを実装している

この章では、抽象クラスとインターフェースの違いを確認し、API リファレンスの `extends` と `implements` を読めるようにします。

## 抽象クラスとは？

抽象クラスとは、そのままではオブジェクトを作れないクラスです。
共通する処理や型をまとめ、サブクラスに具体的な実装を任せるために使われます。

抽象クラスは `abstract` を付けて定義します。

```java
abstract class Animal {
    abstract void makeSound();

    void sleep() {
        System.out.println("眠ります");
    }
}
```

`makeSound` は抽象メソッドです。
抽象メソッドは、メソッド名や引数、戻り値だけを定義し、処理の中身は書きません。

```java
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("ワン");
    }
}
```

`Dog` クラスは `Animal` を継承し、`makeSound` の具体的な処理を実装しています。

```java
Dog dog = new Dog();
dog.makeSound(); // ワン
dog.sleep();     // 眠ります
```

抽象クラスは、共通処理を持ちながら、一部の処理をサブクラスに任せたいときに使います。

## `AbstractList` は抽象クラス

`ArrayList` が継承している `AbstractList` は抽象クラスです。
名前に `Abstract` が付いているクラスは、抽象クラスであることが多いです。

`AbstractList` は、リストとして共通する処理の一部を提供します。
そのうえで、具体的なデータの持ち方や一部の操作は、`ArrayList` や他のサブクラスが実装します。

`ArrayList` は配列を使って要素を管理するリストです。
一方、`LinkedList` は要素同士のつながりを使って管理するリストです。
内部の実装は違いますが、どちらもリストとして扱えます。

```java
List<String> list1 = new ArrayList<>();
List<String> list2 = new LinkedList<>();
```

このように、利用者は「リストとして何ができるか」を意識すればよく、内部の実装方法を必要以上に意識しなくて済みます。

## インターフェースとは？

インターフェースとは、クラスが持つべきメソッドの約束を定義する仕組みです。
インターフェースは `interface` を使って定義します。

```java
interface Printable {
    void print();
}
```

この `Printable` インターフェースは、「`print` メソッドを持つ」という約束を表しています。

クラスがインターフェースを実装するときは、`implements` を使います。

```java
class Report implements Printable {
    @Override
    public void print() {
        System.out.println("レポートを印刷します");
    }
}
```

`Report` クラスは `Printable` を実装しているため、`print` メソッドを持つ必要があります。

```java
Printable printable = new Report();
printable.print(); // レポートを印刷します
```

インターフェースを使うと、「具体的なクラスが何か」ではなく、「どのような操作ができるか」に注目してコードを書けます。

## `List` はインターフェース

`ArrayList` が実装している `List` はインターフェースです。
`List` は、順序を持つコレクションとして必要な操作を定義しています。

たとえば、`List` には次のようなメソッドがあります。

- `add(E e)`
- `get(int index)`
- `remove(int index)`
- `size()`
- `contains(Object o)`

`ArrayList` は `List` を実装しているため、`List` に定義された操作を使えます。

```java
List<String> names = new ArrayList<>();

names.add("Alice");
names.add("Bob");

System.out.println(names.get(0)); // Alice
System.out.println(names.size()); // 2
```

変数の型は `List<String>` ですが、実際のオブジェクトは `ArrayList` です。
このように書くことで、コードは具体的な実装クラスに依存しにくくなります。

## 複数のインターフェースを実装できる

Java のクラスは、直接継承できるクラスは1つだけです。
しかし、インターフェースは複数実装できます。

```java
class Document implements Printable, Comparable<Document> {
    private final String title;

    Document(String title) {
        this.title = title;
    }

    @Override
    public void print() {
        System.out.println(title);
    }

    @Override
    public int compareTo(Document other) {
        return this.title.compareTo(other.title);
    }
}
```

この例では、`Document` クラスが `Printable` と `Comparable<Document>` の2つのインターフェースを実装しています。

`ArrayList` も、`List`、`RandomAccess`、`Cloneable`、`Serializable` という複数のインターフェースを実装しています。

API リファレンスで `implements` の後ろに複数の名前が並んでいたら、そのクラスが複数の約束や性質を持っていると考えればよいです。

## ジェネリクス（総称型）

`ArrayList<E>` や `List<E>` のように、`<...>` を使って型を扱う仕組みをジェネリクス、または総称型と呼びます。
ジェネリクスは、クラスやインターフェースで扱う型を後から指定できるようにする仕組みです。

`ArrayList<E>` の `E` は型パラメータです。
型パラメータは、「ここには後で具体的な型が入る」という仮の名前だと考えると分かりやすいです。

実際に使うときは、`E` の部分に具体的な型を指定します。

```java
ArrayList<String> names = new ArrayList<>();
ArrayList<Integer> scores = new ArrayList<>();
```

`ArrayList<String>` は文字列のリストです。
`ArrayList<Integer>` は整数のリストです。
この `String` や `Integer` は型引数と呼ばれます。

ジェネリクスを使うことで、リストに入れる値の型をコンパイル時にチェックできます。

```java
ArrayList<String> names = new ArrayList<>();
names.add("Alice");
// names.add(100); // コンパイルエラー
```

API リファレンスで `E`、`T`、`K`、`V` などの大文字1文字の型が出てきたら、型パラメータであることが多いです。
一方、実際のコードで `ArrayList<String>` のように指定する `String` は型引数です。

## 抽象クラスとインターフェースの違い

抽象クラスとインターフェースは似ていますが、役割が少し違います。

| 観点 | 抽象クラス | インターフェース |
| --- | --- | --- |
| 目的 | 共通の土台を作る | 操作や性質の約束を定義する |
| キーワード | `abstract class` | `interface` |
| 使い方 | `extends` で継承する | `implements` で実装する |
| 複数指定 | 直接継承は1つ | 複数実装できる |

API リファレンスを読むときは、まず次のように読み分ければ十分です。

- `extends` の後ろにあるもの: 親となるクラス
- `implements` の後ろにあるもの: そのクラスが満たしているインターフェース

## この章のまとめ

この章では、抽象クラスとインターフェースについて学びました。

- 抽象クラスは、共通処理や共通の型をまとめるために使う
- インターフェースは、クラスが持つべき操作や性質の約束を表す
- `extends` は継承を表す
- `implements` はインターフェースの実装を表す
- ジェネリクス（総称型）により、`ArrayList<String>` のように要素の型を指定できる

次の章では、これまで学んだ知識を使って、Java API リファレンスのページ全体をどのように読めばよいかを整理します。
