前回は、Java API リファレンスの概要と、`ArrayList` クラスの基本的な使い方について学びました。
今回は、`ArrayList` クラスを題材に、オブジェクト指向プログラミングの基本概念を学んでいきましょう。

## オブジェクト指向プログラミングとは？

オブジェクト指向プログラミング（Object-Oriented Programming、OOP）とは、プログラムを「オブジェクト」の集まりとして考えるプログラミングの考え方です。

オブジェクトとは、データと処理をひとまとまりにしたものです。
Java では、多くの場合、クラスから作られた実体のことをオブジェクトと呼びます。

たとえば、`ArrayList<String>` のオブジェクトは、次のようなものを持っています。

- データ: リストに入っている文字列
- 処理: 要素を追加する、削除する、取り出す、要素数を返す

```java
ArrayList<String> names = new ArrayList<>();

names.add("Alice");
names.add("Bob");

System.out.println(names.get(0));  // Alice
System.out.println(names.size());  // 2
```

この例では、`names` が `ArrayList` のオブジェクトです。
`names` の中には `"Alice"` や `"Bob"` というデータが入り、`add`、`get`、`size` という処理を呼び出しています。

## オブジェクトは状態と振る舞いを持つ

オブジェクト指向プログラミングでは、オブジェクトを「状態」と「振る舞い」で考えると理解しやすくなります。

`ArrayList` の場合は、次のように考えられます。

| 観点 | `ArrayList` の例 |
| --- | --- |
| 状態 | 現在格納している要素、要素の順番、要素数 |
| 振る舞い | `add` で追加する、`get` で取得する、`remove` で削除する |

コードで見ると、次のようになります。

```java
ArrayList<String> names = new ArrayList<>();

names.add("Alice"); // 状態が変わる
names.add("Bob");   // 状態が変わる

String first = names.get(0); // 状態を使って値を返す
int count = names.size();    // 状態を使って要素数を返す
```

`add` を呼び出すと、`names` の中身が変わります。
`get` や `size` を呼び出すと、`names` の中にある状態をもとに結果が返されます。

## クラスはオブジェクトの設計図

Java では、オブジェクトを作るためにクラスを使います。
クラスは、オブジェクトがどのような状態と振る舞いを持つかを定義する設計図のようなものです。

`ArrayList` はクラスです。
`new ArrayList<>()` と書くことで、`ArrayList` クラスから新しいオブジェクトを作っています。

```java
ArrayList<String> list1 = new ArrayList<>();
ArrayList<String> list2 = new ArrayList<>();

list1.add("Alice");
list2.add("Bob");

System.out.println(list1); // [Alice]
System.out.println(list2); // [Bob]
```

`list1` と `list2` は、どちらも `ArrayList` クラスから作られたオブジェクトです。
しかし、それぞれ別のオブジェクトなので、中身は独立しています。

この「同じクラスから複数のオブジェクトを作れる」という性質は、オブジェクト指向プログラミングの基本です。

## メソッドはオブジェクトに対する操作

Java API リファレンスでは、メソッドの一覧がとても重要です。
メソッドとは、オブジェクトに対して行える操作のことです。

`ArrayList` には、たとえば次のようなメソッドがあります。

| メソッド | 意味 |
| --- | --- |
| `add(E e)` | 要素を追加する |
| `get(int index)` | 指定した位置の要素を取得する |
| `remove(int index)` | 指定した位置の要素を削除する |
| `size()` | 要素数を返す |
| `clear()` | すべての要素を削除する |

API リファレンスを読むときは、まず「このクラスのオブジェクトに何ができるのか」をメソッド一覧から探すとよいでしょう。

## カプセル化とは？

オブジェクト指向プログラミングの重要な考え方に、カプセル化があります。

カプセル化とは、オブジェクトの内部の仕組みを外から直接触らせず、メソッドを通して操作するようにすることです。
Java では、フィールドに `private` を付けることで、クラスの外から直接触れないようにできます。

`ArrayList` の内部では、要素をどのように保存しているのか、容量をどのように増やしているのか、といった処理が行われています。
しかし、使う側は通常、その内部構造を知る必要はありません。

```java
ArrayList<String> names = new ArrayList<>();

names.add("Alice");
names.add("Bob");
names.remove(0);
```

使う側は、`add` や `remove` というメソッドを呼び出すだけです。
内部で配列がどう扱われているかは、`ArrayList` クラスが責任を持ちます。

これにより、使う側のコードはシンプルになります。
また、クラスの内部実装が変わっても、公開されているメソッドの使い方が変わらなければ、使う側のコードを大きく変更せずに済みます。

簡単な例で考えてみましょう。
次の `BankAccount` クラスでは、残高を表す `balance` フィールドを `private` にしています。

```java
class BankAccount {
    private int balance;

    BankAccount(int balance) {
        this.balance = balance;
    }

    void deposit(int amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("入金額は1以上にしてください");
        }
        balance += amount;
    }

    int getBalance() {
        return balance;
    }
}
```

`balance` は `private` なので、クラスの外から直接変更できません。

```java
BankAccount account = new BankAccount(1000);

account.deposit(500);
System.out.println(account.getBalance()); // 1500

// account.balance = -1000; // コンパイルエラー
```

残高を変更したい場合は、`deposit` メソッドを通す必要があります。
そのため、`amount <= 0` のような不正な値をチェックできます。

もし `balance` を外部から直接触れるようにしてしまうと、次のようなコードが書けてしまいます。

```java
class BankAccount {
    int balance;
}

BankAccount account = new BankAccount();
account.balance = -1000;
```

この場合、残高がマイナスになることをクラス自身が防げません。
本来守りたいルールを、使う側のコードすべてに任せることになります。

フィールドを `private` にし、必要な操作だけをメソッドとして公開すると、オブジェクトの状態を安全に保ちやすくなります。
これがカプセル化の大きな目的です。

## API リファレンスとオブジェクト指向の関係

Java API リファレンスは、オブジェクト指向プログラミングの考え方を前提に書かれています。
そのため、次のような言葉が頻繁に出てきます。

- クラス
- オブジェクト
- フィールド
- コンストラクタ
- メソッド
- 継承
- インターフェース

これらの言葉を一つずつ理解していくと、API リファレンスに書かれている内容が読み取れるようになります。

## この章のまとめ

この章では、オブジェクト指向プログラミングの基本的な考え方を学びました。

- オブジェクトは、データと処理をひとまとまりにしたもの
- オブジェクトは、状態と振る舞いを持つ
- クラスは、オブジェクトを作るための設計図
- メソッドは、オブジェクトに対する操作
- カプセル化により、内部の仕組みを隠して安全に操作できる
- フィールドを `private` にすると、外部から不正な状態に変更されることを防ぎやすくなる
